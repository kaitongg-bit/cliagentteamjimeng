# Creator Insights Mapping

这份文档把两位 AI 视频创作者访谈中的洞察，映射到 Agent Team 设计和产品验证。

## 关键结论

创作者真正需要的不是“AI 帮我多写几个 prompt”，而是：

> 把专业创作判断力拆成可复用、可审查、可迭代的流程。

其中最重要的判断力包括：

- 剧本结构判断
- 镜头语言判断
- 视觉风格判断
- 摄影参数判断
- 表演判断
- 剪辑判断
- 结果诊断判断
- 工具路由判断

## 访谈洞察到 Agent 的映射

| 创作者洞察 | 产品/流程含义 | 对应 Agent |
|---|---|---|
| 普通用户不知道“电影感”具体是什么 | 入口不能只让用户写 prompt，要帮用户锁定风格、镜头、光影 | 美术概念设计师、摄影指导 |
| AI 生成 prompt 经常噪音太多 | 需要 prompt 审查，删掉矛盾和无效词 | Prompt Structurer、审片人 |
| 大特写不能同时详细描述背景墙 | 需要专业镜头逻辑校验 | 摄影指导、细分镜师 |
| 人物看哪里会影响电影感 | 表演和镜头不能分开 | AI 演员演技指导、导演 |
| 长视频比 1 分钟视频难得多 | 10 分钟需要剧本、场景、风格、剪辑连续性 | Showrunner、制片人、剪辑 |
| 风格是资产，不应直接公开核心私有 prompt | 产品要区分通用方法和创作者私有风格 | 美术概念设计师、版权伦理审查 |
| 创作者会用多个模型互相审 | Agent Team 的价值在 debate 和 critic loop | 剧本医生、审片人、观众 |
| 生成后“不知道哪里怪”是大痛点 | 需要结果诊断工作流 | 审片人、观众、导演 |

## 产品验证假设

### 假设 1：Agent Team 会提升最终质量

可观测指标：

- 剧本评分提升
- 分镜评分提升
- prompt 可执行性提升
- 首轮可用素材比例提升

### 假设 2：Agent Team 会提升过程可控性

可观测指标：

- 用户能指出为什么某版更好
- 用户能复用 visual bible / shot spec / prompt 模板
- 返工时能明确定位问题类型

### 假设 3：Agent Team 适合作为专业入口

可观测指标：

- 用户愿意为更长等待付出成本
- 用户把它用于 3-10 分钟较高价值项目，而不是随手生成
- 过程产物本身被用户保存或复用

## 研发要验证的真实问题

研发说“验证 Agent Team”，最好翻译为以下四个问题：

1. **效果**：多角色协作是否比单 Agent 明显更好？
2. **归因**：好是因为角色拆分、debate、critic loop，还是只是多花了 token？
3. **成本**：提升是否覆盖延迟、复杂度和生成成本？
4. **入口**：用户是否需要一个显式“AI 制作组/导演组”入口？

## 产品入口建议

不要直接叫 Agent Team。

更适合的用户语言：

- AI 编剧室
- AI 导演组
- 电影级创作模式
- 专业分镜与审片
- 多角色共创
- 让 AI 制作组打磨

建议先作为高级模式进入：

- 10 分钟微电影
- 高质量广告片
- 系列短剧 pilot
- 风格化概念片
- 创作者工作台

不建议一开始放在所有普通生成入口里，因为它会增加等待时间和认知负担。
