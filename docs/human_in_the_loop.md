# Human-in-the-Loop Design

## 总原则

Human 不只是省 token 的替代品，而是 Agent Team 的共同创作者。

Agent 负责：

- 扩展可能性
- 提供专业检查
- 结构化产物
- 找矛盾和盲点
- 把人类审美输入转成可执行规则

Human 负责：

- 提供真实偏好
- 选择方向
- 粘贴私有或半私有风格解析词
- 判断哪个版本真的有感觉
- 最终拍板

## 保留但人类参与的高价值 Agent

### 观众 Agent 必须保留

原因：

- 它能持续模拟“看不看得懂”“想不想继续看”。
- 它可以在剧本、分镜、成片三个阶段都给反馈。
- 它能帮助产品验证过程产物是否提升用户判断力。

Human 的参与：

- 在每轮观众反馈后补一句真实直觉：想看 / 不想看 / 不懂 / 有感觉但不够。
- 对关键镜头和结尾做最终记忆点判断。

### 辩论 Agent 必须保留

原因：

- 创意和剧本需要反方攻击，否则容易自嗨。
- 视觉风格需要有人反对“只是堆词”。
- 结尾需要争论，否则容易落入伪深刻。

Human 的参与：

- 不代替辩论，而是参与辩论。
- Human 可以直接说：这个俗、这个太解释、这个我没感觉、这个方向有可能。
- Showrunner 记录最终取舍。

## 机审一轮，人类拍板

审片人保留一轮机器审查：

- 先把问题分类到剧本、视觉、镜头、表演、剪辑、prompt、工具能力。
- 给出下一轮修改 brief。

Human 再拍板：

- 哪些是真的问题。
- 哪些是 AI 过度挑刺。
- 哪些问题先不改，因为成本太高。

## 美术风格共创

美术 Agent 的主要价值不是替人类找完所有 reference，而是启发人类知道该找什么。

Agent 可以输出：

- 可以参考的电影类型
- 可以搜索的关键词
- 适合的导演/摄影/时代风格
- 色彩和布光方向
- ShotDeck / 电影截图应重点看什么

Human 可以输入：

- 风格 style skill 的解析词
- 参考片名
- 剧照描述
- 自己喜欢和不喜欢的画面
- 私有风格库中的可公开字段

Agent 再负责：

- 清洗成 visual bible。
- 去掉侵权或过度模仿风险。
- 转成镜头、色彩、材质、光影、构图规则。

## 不需要人类逐镜头介入的部分

粗分镜、细分镜不需要人类逐条写。

但多个 Agent 必须参与：

- 编剧：确认每个镜头服务戏剧动作。
- 导演：确认情绪和视点。
- 美术：确认空间、角色、道具和视觉规则。
- 摄影：确认景别、焦段、光线、运动。
- 剪辑：确认节奏和衔接。
- AI 演技指导：确认表演细节可见。
- Prompt Structurer：确认能转成即梦 prompt。

## Token 控制策略

不要砍角色，砍无效轮次。

建议：

- 观众 Agent 每个大阶段一轮，不逐句插话。
- Debate Loop 限制为 1 轮正方、1 轮反方、1 轮人类判断、1 轮 Showrunner 决策。
- 审片 Agent 每次生成后一轮，不做无限追问。
- 美术 Agent 输出参考方向后等待 Human 粘贴 style，再继续结构化。
- Prompt Structurer 对全片做结构化，但只对关键镜头做深度多轮打磨。

## 最小可跑版本

保留角色：

- Showrunner
- 编剧
- 剧本医生
- 美术概念设计师
- 风格策展人
- 导演
- 摄影指导
- 粗分镜师
- 细分镜师
- Prompt Structurer
- Prompt Compliance Reviewer
- 剪辑
- AI 演员演技指导
- 审片人
- 观众 Agent
- Human Creative Director

第一轮不必每个 Agent 都长篇输出。重点是每个关键判断都有角色负责。
